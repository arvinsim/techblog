Reverting changes in Git

January 31, 2015
  • git
  • svn

Git has a number of terms in common with Subversion that actually mean different things. It is something that often trips up new Git users who have used Subversion before.

One of this is how to change your current branch back to the latest commit.

Subversion uses the revert term while Git uses reset

Going back without saving changes

To do a quick reset in Git without saving your current changes, just do this command

git reset {files} ...

If you don’t pass the {files} argument, it will assume that you want to reset everything

Going back while storing the changes

If on the other you want to

git stash
git reset {files} ...
git stash pop
